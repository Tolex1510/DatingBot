# Система рейтинга

## Общее описание

Система рейтинга анкет состоит из трёх уровней, каждый из которых учитывает различные факторы для формирования итогового рейтинга пользователя.

## Уровень 1: Первичный рейтинг

Первичный рейтинг формируется на основе данных из анкеты пользователя и не меняется со временем (только при обновлении анкеты).

### Факторы

| Фактор | Описание | Вес |
|--------|----------|-----|
| Возраст | Возраст пользователя | 0.2 |
| Пол | Пол пользователя | 0.1 |
| Интересы | Перечень интересов | 0.3 |
| География | Город/страна | 0.2 |
| Полнота анкеты | Процент заполнения полей | 0.1 |
| Фотографии | Количество загруженных фото | 0.1 |

### Формула

```
primary_rating = (age_score * 0.2) + (gender_score * 0.1) + 
                 (interests_score * 0.3) + (geo_score * 0.2) + 
                 (completeness_score * 0.1) + (photos_score * 0.1)
```

## Уровень 2: Поведенческий рейтинг

Поведенческий рейтинг динамически корректируется на основе взаимодействия других пользователей с анкетой.

### Факторы

| Фактор | Описание | Вес |
|--------|----------|-----|
| Likes Count | Количество полученных лайков | 0.25 |
| Like/Dislike Ratio | Соотношение лайков и пропусков | 0.25 |
| Match Rate | Частота взаимных лайков (мэтчей) | 0.25 |
| Message Rate | Частота инициирования диалогов | 0.15 |
| Activity Time | Активность в определённое время суток | 0.1 |

### Формула

```
behavioral_rating = (likes_count * 0.25) + (like_dislike_ratio * 0.25) + 
                     (match_rate * 0.25) + (message_rate * 0.15) + 
                     (activity_time * 0.1)
```

## Уровень 3: Комбинированный рейтинг

Интегрирует первичный и поведенческий рейтинги по весовой модели с учётом дополнительных факторов.

### Формула

```
final_rating = (primary_rating * primary_weight) + 
               (behavioral_rating * behavioral_weight) + 
               (bonus_points)
```

### Дополнительные баллы

| Фактор | Описание | Бонус |
|--------|----------|-------|
| Referral | Приглашение друзей | +0.1 за каждого |
| Premium | Премиум подписка | +0.2 |
| Verified | Верифицированный аккаунт | +0.15 |

## Пересчёт рейтингов

### Celery задачи

1. ** hourly_rating_update ** — ежечасный пересчёт поведенческого рейтинга
2. ** daily_rating_update ** — ежедневный полный пересчёт всех рейтингов
3. ** weekly_rating_aggregation ** — еженедельная агрегация статистики

### Redis кэширование

- Предварительно отранжированные списки анкет хранятся в Redis
- При начале сессии пользователя первая анкета проходит полный путь через все сервисы
- Одновременно подгружаются 10 следующих анкет в Redis
- На последней анкете из 10 круг повторяется

## Оптимизация

- Индексы в PostgreSQL для быстрого поиска по рейтингу
- Кэширование в Redis для уменьшения нагрузки на БД
- Асинхронная обработка через RabbitMQ
